## 1. Tab
### 구현한 방법과 이유에 대한 간략한 내용
 - 탭 메뉴 부분은 `<ul>` 엘리먼트로 구현하고 아래에 content 부분을 두고 선택한 `value`에 따라 해당 `className`을 달리줘서 선택된 `value`의 Tab만 보이게 구현했습니다.

### 구현하면서 어려웠던 점과 해결 방법
 - 어떻게 선택되지 않은 탭을 보이지 않게 할지 조금 고민해보다 `hidden` 프로퍼티를 엘리먼트에 주었는데 `flex`의 경우 `hidden` 프로퍼티가 작동하지 않아 `className`을 조건문으로 줘서 `display:none` 스타일을 적용시켰습니다.

### 자세한 실행 방법
 - 상단의 Tab 메뉴 부분을 선택하면 해당 탭에 속한 내용이 아래에 나타납니다.
<br/><br/>

## 2. AutoComplete
### 구현한 방법과 이유에 대한 간략한 내용
- 최상단 `<div>` 엘리먼트 안에 `<input>`과 추천 검색어 부분을 두었고
검색어를 입력하면 `Recommends` state에 추가하여 입력을 할 때마다 해당 검색어와 `Recommends`를 filter하여 아래에 추천 검색어를 나타냈습니다.

### 구현하면서 어려웠던 점과 해결 방법
- 검색어 입력 후 엔터를 하면 검색어를 추천 검색어에 집어 넣는 과정에서 빈 텍스트의 값일 때도 들어가게 되어 조건문을 주어 제외시켰습니다.
- 또 이미 검색했던 검색어는 추천 검색어에 들어가지 않도록 `some` 메서드를 사용하여 존재 여부를 확인하고 존재하면 종료되도록 하였습니다.

### 자세한 실행 방법
- 입력창에 검색어를 입력하면 하단에 추천 검색어가 뜨고 엔터를 누르면 추천 검색어에 존재하지 않는 입력의 경우 추천 검색어에 추가됩니다.
<br/><br/>

## 3. ClickToEdit
### 구현한 방법과 이유에 대한 간략한 내용
- `<input>` 엘리먼트의 `blur` 속성을 사용하여 해당 해당 입력의 focus가 해제된 경우 이벤트를 감지하여 입력값이 하단에 보이도록 구현하였습니다.

### 구현하면서 어려웠던 점과 해결 방법
- `blur` 속성을 생각하지 못해서 `Name`과 `Age` state 2개만 가지고 구현하려다 보니 focus 해제 시 이벤트 발생이 어려워 `blur` 속성을 사용하여 해결하였습니다.

### 자세한 실행 방법
- 이름과 나이 입력창에 입력 후 포커스를 해제하면 하단에 입력한 값이 나타납니다.
<br/><br/>

## 4. Modal
### 구현한 방법과 이유에 대한 간략한 내용
- `index.html`에 backdrop과 modal-root `<div>`를 두고 portal을 사용하여 모달이 최상에 보일 수 있도록 구현하였습니다.

### 구현하면서 어려웠던 점과 해결 방법
- backdrop 부분을 설정하지 않고 뒷배경과 close 이벤트를 구현하기 어려워 backdrop을 따로 두어 해당 부분이 클릭되었을 때 해당 모달이 닫힐 수 있도록 하였습니다.

### 자세한 실행 방법
- `Open Modal` 버튼을 클릭하면 모달이 열리고 모달 상단의 `x`나 backdrop 부분을 클릭하면 해당 모달이 닫힙니다.
<br/><br/>

## 5. Tag
### 구현한 방법과 이유에 대한 간략한 내용
- 최상단에 Wrapper `<div>`를 두고 아래에 입력한 태그가 나타나는 부분과 태그를 추가하는 입력 부분을 두었습니다.
- 입력을 하고 엔터를 누르면 `Tags` state에 추가하고 `Tags`의 `map()` 메서드를 이용하여 추가된 태그들을 출력하였습니다.

### 구현하면서 어려웠던 점과 해결 방법
- 처음에 빈 값과 중복된 태그를 제외시키지 않아 삭제 시 태그 값만을 가지고 태그를 삭제하여 동일한 텍스트 태그들이 다같이 지워졌는데 동일한 태그는 `some()` 메서드를 사용하여 존재하면 제외시켰습니다.

### 자세한 실행 방법
- 입력창에 텍스트를 입력하고 엔터를 누르면 좌측에 해당 태그가 추가되고 추가된 태그의 `x`를 누르면 해당 태그가 삭제됩니다.

<br/><br/>

## 6. Toggle
### 구현한 방법과 이유에 대한 간략한 내용
- `<input>` 엘리먼트의 type을 `checkbox`로 두고 `value` state를 설정하여 해당 값이 체크되면 토글이 켜지고 해제되면 꺼지도록 구현하였습니다. 

### - 구현하면서 어려웠던 점과 해결 방법
- 해당 토글을 어떻게 스타일링 해야할지 어려웠는데 `translate`와 `:checked` 속성을 사용하여 해당 스타일을 적용시켰습니다.

### 자세한 실행 방법
- 토글을 클릭하면 ON/OFF가 되고 그 상태를 하단에 나타냅니다.
